## The 10 Rules of Reliable Data Science

1. Start organized, stay organized
2. Everything comes from somewhere and the raw data is immutable
3. Version control is basic professionalism
4. Notebooks are for exploration, source files are for repetition
5. Tests and sanity checks prevent catastrophes
6. Fail loudly, fail quickly
7. Project runs are fully automated from raw data to final outputs
8. Important parameters are extracted and centralized
9. Project runs are verbose by default and result in tangible artifacts
10. Start with the simplest possible end-to-end pipeline
